# 移动端触摸热区优化

## 背景

用户反馈移动端输入框和快捷栏按钮难以点击触发，原因是可点击区域小于 44px 的最小触摸目标标准。

## 问题分析

### InputBar（命令行输入框）
| 项目 | 修改前 | 修改后 |
|------|--------|--------|
| 容器高度 | 56px | 56px |
| input 高度 | 40px | 100%（充满容器）|
| 上下 padding | 各 8px | 0 |
| **可点击区域** | 40px | 56px |

### CommandPicker（快捷栏按钮）
| 项目 | 修改前 | 修改后 |
|------|--------|--------|
| 行容器高度 | 48px | 48px |
| 按钮高度 | 36px | 100%（充满行容器）|
| 上下 padding | 各 6px | 0 |
| **可点击区域** | 36px | 48px |

## 解决方案

**核心思路**：让可点击元素充满其父容器，消除无效点击区域，同时保持视觉样式不变。

### 修改文件
- `frontend/src/components/input/InputBar.tsx`
- `frontend/src/components/input/CommandPicker.tsx`

## 验收标准

- 点击 InputBar 输入框任意位置都能触发键盘
- 点击快捷栏按钮任意位置都能触发功能
- UI 视觉效果保持不变
